Prurba de convergencia apsoluta

Nota: l primera columna no se tiene en cuenta para el anÃ¡lisis

La funciÃ³n computeH calcula el exponente de Hurst (H) para cada columna. Este valor (entre 0 y 1) mide:

  • H â 0.5: Comportamiento aleatorio (ej: acciones).

  • H > 0.5: Persistencia (ej: tendencias fuertes en ventas).

  • H < 0.5: Anti-persistencia (ej: fluctuaciones errÃ¡ticas).

  • time_trim = 1/3> Recorta el primer tercio, de los datos para reducir el ruido âEn este caso, 1/3 indica que se eliminarÃ¡ el 33% de los puntos en ambos extremos al ajustar el modelo.â

  • Emplea FQSB (mÃ©todo robusto) para corregir autocorrelaciÃ³n y heterocasticidad

Este modelo realiza un contraste entre:

\(Ho\) Las series presentan convergencia absoluta

\(Ha\) Las series no presentan convergencia absoluta

# Paso 1: Calculamos el exponente de Hurst
H <- computeH(log_wide[,-1],quantity = "H")

# Paso 2: Estimamos el modelo de convergencia.
round(estimateMod(H,time_trim = 1/3, HACmethod = "FQSB"),3)

  • Dado su \(Pâvalue\) se rechaza la hipÃ³tesis nula, por lo tanto, se puede afirmar estadÃ­sticamente que no existe convergencia absoluta
